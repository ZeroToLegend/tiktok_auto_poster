# TikTok Auto Poster — Agent Instructions

Compatible with: Claude Code, Cursor, Codex, Gemini CLI, Windsurf.

This is a **multi-skill agent system** for automating TikTok posts with safety guardrails.

## Quick start for agents

1. **Read `CLAUDE.md`** — it contains orchestration rules (applies to all agents despite name).
2. **Load skills from `.agents/skills/`** (or `.claude/skills/` as fallback).
3. **Start every pipeline** by checking `.agents/tiktok-context.md` exists.
4. **Use Bash tool** to call `scripts/tool_*.py` — each tool returns JSON.

## Skill catalog

| Skill | Purpose | When to invoke |
|---|---|---|
| `tiktok-context` | Capture creator context | First time, missing context file |
| `tiktok-video-prep` | Process video to 9:16 + normalize | Have raw video file |
| `tiktok-hook-writer` | Craft opening 5 words | Before caption |
| `tiktok-caption-writer` | Write full caption body | After hook chosen |
| `tiktok-hashtag-strategy` | Select 3-5 tags (pyramid) | Before upload |
| `tiktok-scheduler` | Plan posting time | User mentions schedule |
| `tiktok-uploader` | Push via official API | API access available |
| `tiktok-advisory-mode` | Manual-post package | No API / fallback |
| `tiktok-analyzer` | Performance insights | User asks for report |

## Critical rules

- NEVER use Selenium, Playwright, or any browser automation — violates TikTok ToS
- ALWAYS check duplicate (video hash) before uploading
- MAX 6 posts/day per account
- Fall back to `tiktok-advisory-mode` if no API access

## File layout (agent-agnostic)

## Tool invocation pattern

All tools return JSON to stdout, exit 0 on success:

```bash
# Process video
python scripts/tool_process_video.py --input=video.mp4 --action=prepare
# → {"processed_path": "...", "hash": "...", "size_mb": 23.4}

# Check quota before scheduling
python scripts/tool_schedule.py check-quota --account=default
# → {"posted_today": 2, "remaining": 4, "can_post": true}

# Get next optimal slot
python scripts/tool_schedule.py next-slot --account=default
# → {"iso": "2026-04-18T20:00:00", "human": "2026-04-18 20:00"}
```

## Error handling

If a tool returns non-zero exit:
1. Parse stderr for error context
2. Match against skill's error table
3. Either retry (if transient) or fall back to advisory-mode

## Extending

To add a new skill:
1. Create `.agents/skills/<name>/SKILL.md` with YAML frontmatter
2. Update this file's catalog
3. Run `./validate-skills.sh` to check format
4. If skill needs Python tool, add `scripts/tool_<name>.py`
